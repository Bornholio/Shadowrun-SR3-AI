# Cyberdecks
*Malice Family — SR3 Campaign*
*File location: RULES/Cyberdecks.md*

---

Cyberdecks are the tools that deckers use to interface with the Matrix. In a nutshell, a cyberdeck is an extremely powerful microcomputer cranking out enough processing power to implement the ASIST interface, converting the decker's neural impulses into holographic command instructions. Running the hottest utilities on the street, the cyberdeck gives the decker the ability to dance across the electron sky.

All decks consist of certain components, fine-tuned by the decker to present the most wiz icon possible. These components generate the decker's persona and define the decker's ratings in the Matrix.

Only one decker can jack into a cyberdeck at any given time. The decker must be physically connected to the deck through a fiber-optic jack to his datajack. The deck must be physically connected to a jackpoint to use the Matrix.

---

## Deck Ratings

The power of a decker's persona is defined by the processing power of his deck's MPCP (Master Persona Control Program), and his Bod, Sensor, Evasion, and Masking programs. The MPCP represents the master operating system for the deck and has an MPCP Rating that measures its ability to take damage and continue functioning. The Bod, Sensor, Evasion, and Masking programs are called **persona programs**.

The numeric ratings of these programs serve as the "Attributes" for the decker's persona and are used whenever tests are made against the decker while in the Matrix.

**MPCP constraints:**
- MPCP Rating × 3 = maximum total of all persona programs combined
- No single Persona Rating may exceed the MPCP Rating
- Maximum utility program rating = MPCP Rating

**Shorthand format:** `MPCP Rating / Bod / Evasion / Masking / Sensor`

*Example — a deck with MPCP-8 and persona programs distributed equally across the maximum total (3 × 8 = 24):*
`MPCP-8/6/6/6/6`

*If the decker increased Bod by 2 (to the maximum of 8), reducing Evasion and Sensor by 1 each:*
`MPCP-8/8/5/6/5`

## Hardening

Hardening represents internal deck programs specifically designed to reinforce the deck's resistance to invasive code such as viruses, gray and black IC.

- For every point of Hardening, reduce the Power of any Damage from Black IC to the on-line icon or the actual decker by 1 for Resistance Tests
- If the icon has been crashed by Gray IC and it makes an Attack Test, add 1 to the Target Number for the Attack Test per point of Hardening
- Hardening works against the Black Hammer and Killjoy utilities, but **not** against other attack utilities

---

## Active Memory

Active memory is the cyberdeck's RAM. A deck's active memory limits the utility programs the deck can run at any one time. For each Mp of active memory, the deck can have the equivalent Mp in utilities loaded simultaneously.

*Example — a deck with 200 Mp of active memory can run no more than 200 Mp of utilities at any one time.*

---

## Storage Memory

Storage memory is analogous to a hard drive. Any program in a deck's storage memory can be loaded onto the deck by using the Swap Memory operation.

All utilities must be kept in storage memory regardless of whether they are currently in active memory. Storage memory is also used for data uploads and downloads. The total Mp of all utilities and other stored data cannot exceed the deck's storage memory.

*Example — a decker with only 500 Mp of storage memory remaining cannot download any file over 500 Mp.*

---

## I/O Speed

I/O Speed governs all uploads and downloads, measured in Mp per Combat Turn. All transfers run at the deck's full I/O speed.

---

## Response Increase

Response Increase is the Matrix equivalent of wired reflexes. Each point of Response Increase increases a persona's Reaction Attribute by 2 and Initiative by +1D6.

- Maximum Response Increase: 3 points
- Response Increase cannot exceed MPCP Rating ÷ 4 (round down)
- A deck with MPCP Rating 3 or below cannot sustain any Response Increase

---

## Detection Factor

The gamemaster uses the decker's Detection Factor as the target number when making tests to detect the decker's presence or prevent him from performing actions within the Matrix.

**Calculation:** Average (round up) of Masking Rating and Sleaze program rating.

*Example — MPCP-8/6/8/6/4 deck running Sleaze-8: (6 + 8) ÷ 2 = **7***

If a decker is not running a Sleaze program, the Detection Factor equals half the Masking Rating. This makes a high-rated Sleaze program a necessity for any decker with aspirations.

---

## The Hacking Pool

The Hacking Pool follows all normal dice pool rules. To determine a decker's Hacking Pool:

> (Intelligence Rating + MPCP Rating) ÷ 3, round down

Any increases to Intelligence apply to the Hacking Pool as well, whether from cyberware or magic. Increases are cumulative.

Hacking Pool dice may generally be added to any test made in the Matrix — System Tests, attack or defense tests, maneuvers, or Attribute Tests.

**Exception:** Hacking Pool dice **cannot** be used in Body or Willpower Tests made to resist the effects of gray or black IC that is damaging the decker. Only Karma Pool dice, enhancements connected to the cyberdeck, or magic boosts to Body or Willpower can help in such situations.

---

## Cyberterminals

Not everyone in the Matrix uses a cyberdeck. Most Matrix users jack in through a cyberterminal — known by deckers as **tortoises** for their lack of speed and finesse. Cyberterminals are simply keyboards with viewing screens and a set of electrodes or a datajack plug.

**Constraints:**
- No cyberterminal can have an MPCP higher than 4
- Cannot be outfitted with Response Increase
- All programs run on them have their ratings reduced by 1

**Advantages:**
- Tortoise users cannot be hurt by black IC or dump shock
- Generally cost 10% of the price of an equivalent cyberdeck

---

## Accessories

Cyberdecks and cyberterminals frequently come with accessories such as off-line storage or a vid-screen display so others may shoulder-surf the Matrix from the decker's point of view.

**Hitcher jacks** (electrode or datajack feed) allow others to jack in and shoulder-surf directly as the decker's icon. Hitchers:
- Cannot manipulate or affect the decker's persona in any way — they are along for the ride only
- Are protected from IC side-effects in the same way as cyberterminal users
- Cannot control the movements or perspective of the decker
