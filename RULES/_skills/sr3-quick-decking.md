# SR3 Quick Decking — GM Reference

Use when a Matrix run comes up mid-session and you don't want to halt the table for full rules. Two options: single test (fastest) or series of tests (more detail).

Regardless of which method you use — describe what happens. System sculpting, IC personas, brief flavor. Don't just read the dice.

---

## Option 1: Single Test

**When to use:** No time at all, or a milk run the decker can clearly handle.

**Roll:** Computer (Decking) + all Hacking Pool
**TN:** System Security Value

**Threshold — minimum successes to get anything:**

| Security Code | Min Successes |
|---------------|--------------|
| Blue | 1 |
| Green | 2 |
| Orange | 3 |
| Red | 4 |

Each net success over threshold = one item from a pre-made list, or one answered question.

**Optional damage check:** System rolls Security Value vs. decker's Detection Factor. Each system success = 1 box persona damage (or MPCP damage, or physical via black IC).

---

## Option 2: Series of Tests

**When to use:** Want more texture, want to give the decker something to work against.

**Roll:** Computer (Decking) only — Hacking Pool not added to dice
**TN:** Security Value (all non-Fight/Escape actions)

**Hacking Pool spending (any point):**
- 1 point → 1 automatic success on any test
- 1 point → automatically suppress 1 IC
- Spent pool does not refresh until end of run

**System Pool (opposes decker):**

| Security Code | System Pool |
|---------------|------------|
| Blue | 1 |
| Green | 2 |
| Orange | 3 |
| Red | 4 |

Spend to negate decker's successes, or give IC an automatic success. Does not refresh until end of run.

### Action Types

| Action | Roll Against | Failure Result |
|--------|-------------|----------------|
| Access | Security Value | Cannot log on |
| Search | Security Value | Alert triggered → IC deployed |
| Control | Security Value | Alert triggered → IC deployed |
| Fight | IC Rating (Opposed) | Net successes = damage boxes |
| Escape | IC Rating (Opposed) | Failure = IC attacks |

**5 alerts = system shutdown.**
**While fighting IC:** Decker can only suppress, escape, or continue fighting — no other actions.

**Damage by IC type:**
- Gray IC → persona damage
- Black IC → physical damage to decker
- Trace IC → traced at 10 net successes (no effect before that)

### TN Modifiers

| Situation | Modifier |
|-----------|---------|
| Missing appropriate utility | +2 |
| First alert triggered | +1 |
| Second alert triggered | +2 |
| Each suppressed IC still active | +1 per IC |

### Quick Difficulty Gauge

- **Blue/Green:** Single test. Competent decker handles this without breaking a sweat.
- **Orange:** Comfortable with pool. Series of tests worth considering for texture.
- **Red:** Run full rules. This is a planned operation.
